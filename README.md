# КУГИС — генератор схем расположения и ортофотопланов

Статическое веб-приложение (без сборки и сервера) для быстрой подготовки материалов
к градостроительным проектам по Москве:

- **Схема расположения** — два панно по образцу: Москва с округами (выбранный округ
  розовым) и округ с районами (выбранный район розовым), точки, пунктирные линии
  и рамка «рассматриваемая территория». Экспорт в PNG (до 4800 px) и SVG
  (редактируется в Illustrator/Inkscape).
- **Ортофотоплан** — спутниковая подложка Esri World Imagery, осветляющая маска вокруг
  выбранного района, граница рассматриваемой территории (рисуется мышью или
  загружается из GeoJSON) с автоматическим подсчётом площади в га, подписи улиц
  (с поворотом), районов/округов, станции метро с бейджами линий (D1–D4, МЦК),
  выноски-рамки. Экспорт в PNG высокого разрешения (сшивка тайлов).

Все подписи перетаскиваются мышью. Состояние ортофотоплана автосохраняется
в браузере (localStorage).

## Данные

`data/*.geojson` сгенерированы из шейпфайлов проекта QGIS
(`_ШАБЛОНЫ/КУГИС/Проект`, местная СК Москвы → WGS84):

- `okruga.geojson` — границы 12 административных округов;
- `rayony.geojson` — 134 района/поселения с привязкой к округу;
- `stations.geojson` — станции МЦК и МЦД-2.

## Локальный запуск

Из-за `fetch()` данных нужен любой статический сервер:

```
cd kugis-schemes
python -m http.server 8000
# открыть http://localhost:8000
```

## Деплой на GitHub Pages

1. Создайте репозиторий на GitHub (например, `kugis-schemes`).
2. В этой папке:

   ```
   git remote add origin https://github.com/<ваш-логин>/kugis-schemes.git
   git push -u origin main
   ```

3. На GitHub: **Settings → Pages → Source: Deploy from a branch → Branch: main / (root) → Save**.
4. Через минуту приложение будет доступно по адресу
   `https://<ваш-логин>.github.io/kugis-schemes/`.

## Подложка

Спутниковые снимки — сервис Esri World Imagery
(Esri, Maxar, Earthstar Geographics). Атрибуция автоматически добавляется
на карту и в экспортируемый PNG.
