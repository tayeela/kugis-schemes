# КУГИС — генератор схем расположения и ортофотопланов

Статическое веб-приложение (без сборки и сервера) для быстрой подготовки материалов
к градостроительным проектам по Москве:

- **Схема расположения** — два панно по образцу: Москва с округами (выбранный округ
  розовым) и округ с районами (выбранный район розовым), точки, пунктирные линии
  и рамка «рассматриваемая территория». Подписи районов автоматически размещаются
  внутри своих полигонов горизонтально (полюс недоступности) с автоподбором кегля.
  Экспорт в PNG (до 4800 px) и SVG (редактируется в Illustrator/Inkscape).
- **Ортофотоплан** — спутниковая подложка на выбор: **Яндекс Спутник** (свежие
  снимки, EPSG:3395), **Esri World Imagery** (Maxar) или **Sentinel-2 cloudless
  2024** (EOX). Осветляющая маска вокруг выбранного района, **дороги и основные
  ходы железных дорог из OSM** (Overpass, кремовые улицы + чёрно-белая штриховка ЖД,
  как в образце), граница рассматриваемой территории (рисуется мышью или загружается
  из GeoJSON) с автоматическим подсчётом площади в га, подписи улиц (с поворотом),
  районов/округов, **станции метро с фирменными иконками** (лого М, кружки линий
  1–18, параллелограммы МЦД D1–D5, МЦК), выноски со светло-коричневой рамкой
  (цвет настраивается). Экспорт в PNG высокого разрешения (сшивка тайлов,
  для Яндекса — эллиптическая проекция).

Шрифт — Century Gothic (если установлен; иначе веб-фолбэк Didact Gothic).
Все подписи перетаскиваются мышью. Состояние ортофотоплана автосохраняется
в браузере (localStorage).

## Данные

`data/*.geojson` сгенерированы из шейпфайлов проекта QGIS
(`_ШАБЛОНЫ/КУГИС/Проект`, местная СК Москвы → WGS84):

- `okruga.geojson` — границы 12 административных округов;
- `rayony.geojson` — 134 района/поселения с привязкой к округу;
- `stations.geojson` — станции МЦК и МЦД-2.

## Локальный запуск

Из-за `fetch()` данных нужен любой статический сервер:

```
cd kugis-schemes
python -m http.server 8000
# открыть http://localhost:8000
```

## Деплой на GitHub Pages

1. Создайте репозиторий на GitHub (например, `kugis-schemes`).
2. В этой папке:

   ```
   git remote add origin https://github.com/<ваш-логин>/kugis-schemes.git
   git push -u origin main
   ```

3. На GitHub: **Settings → Pages → Source: Deploy from a branch → Branch: main / (root) → Save**.
4. Через минуту приложение будет доступно по адресу
   `https://<ваш-логин>.github.io/kugis-schemes/`.

## Подложки и данные

- **Яндекс Спутник** — тайлы core-sat.maps.yandex.net (проекция EPSG:3395;
  неофициальное использование, вне API Яндекса — на усмотрение пользователя).
- **Esri World Imagery** — Esri, Maxar, Earthstar Geographics.
- **Sentinel-2 cloudless 2024** — EOX IT Services, CC BY 4.0 (обзорная, до z14).
- **Дороги** — © участники OpenStreetMap, загрузка через Overpass API
  по границе выбранного района с буфером (кэшируется на сессию).
- **Железные дороги** — генерализованные магистральные ходы (usage=main),
  предсобраны скриптом `tools/fetch_railways.py` в `data/railways.geojson`:
  сегменты склеены в непрерывные линии, параллельные пути и направления
  слиты в один ход, кривые сглажены — штриховка ровная, без «пучков»
  станционных путей.
- **Леса и вода** — natural=wood/landuse=forest и natural=water из OSM,
  предсобраны скриптом `tools/fetch_nature.py` в `data/nature.geojson`
  (упрощение ~22 м, обрезка по границе Москвы). Используются только на схеме
  расположения (в границах выбранного округа); ортофотоплан — чистый снимок.
- **Иконки метро/МЦД** — стандартные значки линий (assets/metro).

Атрибуция подложки автоматически добавляется на карту и в экспортируемый PNG.
